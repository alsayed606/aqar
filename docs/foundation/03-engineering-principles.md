# ٣) المبادئ الهندسية — Engineering Principles
*عقار (Aqar) · وثيقة تأسيسية ٣ من ٧ · مُعتمدة · قواعد تقنية دائمة غير قابلة للانتهاك*

> **الوسم:** **[مطبَّق]** مفروض اليوم · **[يُعتمد الآن]** قاعدة جديدة تسدّ فجوة رصدتها المراجعة. مخالفة أي قاعدة عيبٌ يُردّ في المراجعة. تُستشهد بالأرقام (هـ-NN).

## أ) المعمار
- **هـ-١** منطق الأعمال في قاعدة البيانات (دوال معاملاتية مغطّاة بـ RLS)؛ التطبيق ينسّق ويتحقّق عند الحدود. **[مطبَّق]**
- **هـ-٢** كل وصول للبيانات من الخادم (RSC + Server Actions)؛ لا مفاتيح/منطق حسّاس في المتصفّح. **[مطبَّق]**
- **هـ-٣** سياق المنظمة عبر ترويسة `x-active-org` (request-scoped)، لا في الـ JWT، ويُعاد إثباته كل استعلام. **[مطبَّق]**
- **هـ-٤** الهجرات مصدر الحقيقة؛ `schema_all.sql` يُولَّد لا يُحرَّر يدوياً. **[يُعتمد الآن]**
- **هـ-٥** البوابات سطح منفصل يقرأ عبر `SECURITY DEFINER` محكوم بالهوية؛ RLS المكتب لا يُمسّ. **[مطبَّق]**
- **هـ-٦** لا طبقة API عامة إلا عند حاجة مُثبَتة، وتُبنى فوق دوال DB. **[يُعتمد الآن]**

## ب) قاعدة البيانات
- **هـ-٧** `org_id` + RLS مُفعّلة + سياسة لكل جدول تشغيلي. **[مطبَّق]**
- **هـ-٨** المال `bigint` بالهللات، لا `float`. **[مطبَّق]**
- **هـ-٩** الحالة المالية تُشتق عبر views (`security_invoker`)، لا تُخزَّن. **[مطبَّق]**
- **هـ-١٠** حذف soft دائماً (`deleted_at/by/reason`). **[مطبَّق]**
- **هـ-١١** العقد مجمّد بعد التفعيل؛ التغيير عبر ملحق أو عقد لاحق. **[مطبَّق]**
- **هـ-١٢** ترقيم متسلسل بلا فجوات لكل (منظمة، سنة) عبر عدّاد ذرّي. **[مطبَّق]**
- **هـ-١٣** تخزين `timestamptz` (UTC)؛ عرض Riyadh؛ هجري نصّي للعرض. **[مطبَّق]**
- **هـ-١٤** `enum` للمفردات المغلقة؛ `org_type` عرض فقط (لا فرع عليه). **[مطبَّق]**
- **هـ-١٥** الهجرات idempotent. **[مطبَّق]**

## ج) الأمان
- **هـ-١٦** RLS حدّ العزل؛ يُثبَت لكل استعلام (`has_org_access`)، لا يُوثَق بالواجهة. **[مطبَّق]**
- **هـ-١٧** ربط الطرف بالهوية عبر مسار الدعوة فقط (حارس `allow_party_link`)؛ لا ربط تلقائي بالجوال. **[مطبَّق]**
- **هـ-١٨** رموز الدعوة/البوابة مُجزّأة (sha256)، أحادية الاستخدام، منتهية الصلاحية، تُعرض مرّة. **[مطبَّق]**
- **هـ-١٩** لا أسرار تُلتزَم؛ `anon` عام مقبول؛ `service_role` ممنوع في `NEXT_PUBLIC_*`. **[مطبَّق]**
- **هـ-٢٠** كل دالة `SECURITY DEFINER` تثبّت `search_path` وتُحكَم (هوية/مسؤول) ولها اختبار عزل مُلتزَم. **[يُعتمد الآن]**
- **هـ-٢١** التحقّق عند الحدود؛ كل محتوى خارجي بيانات لا أوامر؛ لا PII في روابط/معاملات URL. **[مطبَّق]**
- **هـ-٢٢** حماية الوصول (Deployment Protection) مُفعّلة قبل بيانات حقيقية + حدّ معدّل على الدخول. **[يُعتمد الآن]**

## د) سلامة البيانات المالية
- **هـ-٢٣** التخصيص لا يتجاوز الدفعة ولا الاستحقاق (`tg_allocation_check`). **[مطبَّق]**
- **هـ-٢٤** الضريبة بحسب النوع (سكني معفى / تجاري ١٥٪) على كل استحقاق. **[مطبَّق]**
- **هـ-٢٥** تقسيم متساوٍ مع باقي القسمة على آخر استحقاق؛ لا خسارة/زيادة هللة. **[مطبَّق]**
- **هـ-٢٦** كل مستند مالي مرقّم قابل للطباعة ومطابق لمصدره (snapshot). **[مطبَّق]**

## هـ) التدقيق
- **هـ-٢٧** كل عملية حسّاسة تُسجَّل عبر `write_audit` بثلاثة معرّفات (org, identity, membership). **[مطبَّق]**
- **هـ-٢٨** سجل التدقيق append-only — لا `UPDATE`/`DELETE`. **[مطبَّق]**
- **هـ-٢٩** الحالة قابلة لإعادة البناء لأي تاريخ سابق. **[مطبَّق]**

## و) معايير الكود
- **هـ-٣٠** TypeScript، ملفات < ٥٠٠ سطر، مطابقة أسلوب الجار، تسمية تكشف النيّة. **[مطبَّق]**
- **هـ-٣١** `clean-code-guard` مرور حراسة قبل التسليم؛ لا كود ميت، لا أعلام تخمينية. **[مطبَّق]**
- **هـ-٣٢** إعادة استخدام مساعدي `lib/` (money, num, dates, rows, labels, tafqit, zatca). **[مطبَّق]**
- **هـ-٣٣** Server Actions تتحقّق عند الحدود وتُرجِع أخطاء عربية ودّية. **[مطبَّق]**
- **هـ-٣٤** لا التزام لملفات أدوات Ruflo (`CLAUDE.md`, `.claude-flow/`, `.mcp.json`, `.swarm/`, `ruvector.db`). **[يُعتمد الآن]**
- **هـ-٣٥** الكود الميت يُزال قبل التسليم (مثال: OTP المخصّص 0004/0014 المهجور). **[يُعتمد الآن]**

## ز) الاختبار
- **هـ-٣٦** كل دالة DB ذات أثر مالي أو عزلي لها اختبار مُلتزَم في `supabase/tests/` — لا سكربتات مؤقتة. **[يُعتمد الآن]**
- **هـ-٣٧** اختبار عزل إلزامي لأي سطح قراءة جديد يواجه مستأجراً. **[يُعتمد الآن]**
- **هـ-٣٨** التحقّق على PostgreSQL 17 (المشغّل المُضمّن) + `tsc` + `build` نظيفان قبل الدمج. **[مطبَّق يدوياً]**
- **هـ-٣٩** CI يشغّل السويت المُلتزَم على كل دفع. **[يُعتمد الآن]**
- **هـ-٤٠** «مُتحقَّق» = اختبار مُلتزَم + `tsc` + `build` (لا «نجح عندي»). **[يُعتمد الآن]**
