# ٤) تعريف المجال — Domain Definition
*عقار (Aqar) · وثيقة تأسيسية ٤ من ٧ · مُعتمدة · لغة النظام المشتركة*

> **قاعدة اللغة:** نستخدم هذه المصطلحات حصراً في الكود والتوثيق. كل كيان موسوم **[موجود]** أو **[مُخطَّط]**.

## خريطة العلاقات
```
Identity (شخص عالمي / تسجيل الدخول)
   └─(Membership: هوية × منظمة + دور + نطاق)──> Organization (المكتب)
Party (طرف داخل المنظمة) ──[identity_id?]──> (المالك/المستأجر في البوابة = Party مرتبط بهوية، وليس Membership)
     ├─ Owner (فرع دور)   └─ Tenant (فرع دور)
Organization → Property → (Building?) → Unit → UnitStatusHistory
     │            ManagementAgreement (مالك × عقار: نموذج الأتعاب)
Contract (وحدة × مستأجر) → Charge (استحقاق) ──< PaymentAllocation >── Payment (دفعة)
     ├─ ContractAmendment (ملحق)   └─ renewed_from (عقد لاحق/تجديد)   └─> Invoice → InvoiceLine / Credit·Debit Note
OwnerRemittance (توريد للمالك) · AuditLog · Document · ImportBatch/Row · org_counter (الترقيم)
```

## الهوية والوصول
- **Identity — الهوية** `app.identity` **[موجود]:** الشخص عالمياً (`id=auth.uid()`، مفتاحه الجوال). لا يحمل `org_id` ولا صلاحيات بذاته.
- **Membership — العضوية** `app.membership` **[موجود]** *(= «الموظّف»)*: انتماء هوية لمنظمة بدور ونطاق. تخصّ موظّفي المكتب فقط؛ مستخدمو البوابة ليسوا أعضاء.
- **Invitation — الدعوة** `app.invitation` **[موجود]:** عضوية أو ربط بوابة؛ رمز مُجزّأ أحادي الاستخدام؛ يُقبل بالرمز لا بالجوال.
- **Party — الطرف** `app.party` **[موجود]:** من الشخص/الكيان في سجلات هذه المنظمة؛ يوحّد المالك/المستأجر/المورّد/الوسيط؛ محلّي داخل المنظمة.

## الأصول
- **Organization — المكتب** `app.organization` **[موجود]:** المستأجر (tenant)؛ جذر العزل؛ `org_type` عرض فقط.
- **Property — العقار** `app.property` **[موجود]:** قمة الهرمية؛ يملكه مالك دائماً؛ وحدة العزل الثانية.
- **Building — المبنى** `app.building` **[موجود]:** طبقة وسطى اختيارية (بلا واجهة كاملة).
- **Unit — الوحدة** `app.unit` **[موجود]:** الأصل القابل للإيجار؛ «عقد نشط واحد لكل وحدة»؛ الحالة تُشتق من التاريخ.
- **UnitStatusHistory** `app.unit_status_history` **[موجود]:** مقاطع زمنية؛ مصدر حساب الإشغال.

## الأطراف وأدوارها
- **Owner — المالك** `app.owner` **[موجود]:** فرع دور؛ `is_self=true` = المنظمة تملك نفسها؛ يحمل الآيبان/الضريبة؛ متلقّي التوريد والكشف.
- **Tenant — المستأجر** `app.tenant` **[موجود]:** فرع دور؛ طرف العقد؛ مصدر الدفعات.
- **Broker/Vendor** *(دور محجوز، بلا ميزات)* **[مُخطَّط جزئياً]**.

## العلاقة التعاقدية والإدارية
- **Contract — العقد** `app.contract` **[موجود]:** وثيقة الإيجار؛ مجمّد بعد التفعيل؛ لا حالة مالية مخزّنة؛ `renewed_from_contract_id` للتجديد.
- **ContractAmendment — الملحق** `app.contract_amendment` **[موجود]:** السبيل الوحيد لتغيير عقد نشط؛ مُصدَّر بإصدار وحمولة before/after.
- **ManagementAgreement — اتفاقية الإدارة** `app.management_agreement` **[موجود]:** رابط زمني مالك↔عقار يحمل نموذج الأتعاب — أساس كشف المالك؛ مستقل عن العقد.

## المالية
- **Charge — الاستحقاق** `app.charge` **[موجود]** ⭐: مبلغ مستحق بتاريخ؛ أساس اشتقاق الحالة؛ **داخلي، ليس مستنداً ضريبياً**.
- **Payment — الدفعة** `app.payment` **[موجود]:** مبلغ مستلم بطريقة؛ يحمل رقم سند قبض؛ الفائض رصيد غير مخصّص.
- **PaymentAllocation — التخصيص** `app.payment_allocation` **[موجود]:** يربط دفعة باستحقاق؛ لا يتجاوز أيّهما؛ الجسر الذي يجعل الحالة مشتقّة.
- **Invoice — الفاتورة الضريبية** `app.invoice`(+`invoice_line`) **[موجود]:** مستند ZATCA (لقطة + QR)؛ `doc_kind` (فاتورة/إشعار دائن/مدين)؛ تُصدَر عند نقطة الاستحقاق الضريبي بصرف النظر عن السداد.
- **OwnerRemittance — توريد المالك** `app.owner_remittance` **[موجود]:** صرف فعلي للمالك (سند صرف مرقّم).
- **Counter — العدّاد** `app.org_counter`/`next_counter` **[موجود]:** ترقيم ذرّي لكل (منظمة، سنة، نوع).

## الحوكمة والبيانات
- **AuditLog** `app.audit_log` **[موجود]:** append-only بثلاثة معرّفات.
- **Document** `app.document` **[موجود كجدول / بلا واجهة]:** مرفقات (صك/عقد/هوية) — لا رفع/عرض بعد.
- **ImportBatch/Row** **[موجود]:** تخزين مرحلي للاستيراد مع تحقّق/اعتماد/تراجع.
- **FeatureFlag** `app.feature_flag` **[موجود]:** فروق المنظمات (بدل الفرع على `org_type`).

## كيانات مُخطَّطة (لغة محجوزة)
Maintenance (الصيانة) · Task (المهمة) · Notification (الإشعار) · Subscription/Plan (الاشتراك/الخطة) · PaymentGatewayTxn (معاملة بوابة الدفع) — كلها **[مُخطَّط]**.

## تمييزان حاسمان
1. **الموظّف (Membership) ≠ مستخدم البوابة (Party مرتبط بهوية).**
2. **Charge (داخلي) ≠ Invoice (ضريبي) ≠ سند القبض على Payment (إثبات دفع).**
