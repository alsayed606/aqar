# 4) الأدوار والصلاحيات

عند إضافة عضو إلى منشأتك، تحدّد له **دوراً** يضبط ما يمكنه رؤيته وعمله. الأدوار الحالية:

| الدور | الوصف |
|---|---|
| **مالك** | أعلى صلاحية. ينشئ المنشأة، يدير الأعضاء والصلاحيات، ويصل لكل شيء. (لا يمكن إزالة آخر مالك في المنشأة.) |
| **مدير** | إدارة كاملة للعقارات والعقود والأعضاء ضمن المنشأة. |
| **مدير محفظة** | إدارة مجموعة عقارات محدّدة (نطاق) دون بقية المنشأة. |
| **محاسب** | التركيز على الاستحقاقات والدفعات والتقارير المالية. |
| **موظف** | إدخال ومتابعة البيانات اليومية. |
| **مطّلع** | اطّلاع فقط دون تعديل. |

## نطاق العقارات
يمكن تقييد بعض الأعضاء (مثل مدير المحفظة) بـ **مجموعة عقارات بعينها** داخل المنشأة، فلا يرون غيرها. هذه طبقة حماية ثانية فوق عزل المنشآت.

## عزل البيانات (مهم)
- كل منشأة معزولة تماماً عن غيرها؛ لا تسريب بين عملاء متنافسين.
- إلغاء عضوية أي شخص **يقطع وصوله فوراً**.
- كل عملية حسّاسة تُسجَّل في **سجل تدقيق** غير قابل للتعديل (من فعل ماذا، في أي منشأة، وبأي صلاحية وقتها).

## إجراءات حسّاسة (تتطلب تحقّقاً إضافياً)
⏳ **قريباً:** بعض الإجراءات ستطلب رمز تحقق إضافياً (Step-up) لحمايتك، مثل:
- تغيير رقم الجوال.
- إدارة الأعضاء والصلاحيات.
- تعديل البيانات البنكية (الآيبان) للمالك.
- تصدير البيانات.

---

## دعوة عضو (قريباً)
سيتمكّن المالك/المدير من إرسال دعوة عبر رقم جوال العضو، مع تحديد دوره ونطاقه. الدعوة تُقبل برمز خاص، وتُنشأ العضوية فقط عند قبول الدعوة.
