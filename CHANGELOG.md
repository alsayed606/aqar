## 2026-07-22

### ميزة — استيراد Excel (رفع ← تحقّق ← معاينة ← اعتماد ← تراجع)
- **`/app/import`**: تنزيل قوالب عربية جاهزة (عقارات/وحدات/ملّاك/مستأجرون/عقود/استحقاقات)، ورفع الملف بعد تعبئته.
- تحليل الملف داخل خادم Next (SheetJS)، وتخزين الصفوف، والتحقق عبر `import_validate` في القاعدة.
- **`/app/import/[batchId]`**: معاينة النتيجة (صحيحة/أخطاء)، **تقرير أخطاء لكل صف** (الصف + الحقل + السبب)، واعتماد الصفوف الصحيحة (`import_commit`)، والتراجع عن الدفعة كاملة (`import_revert`).
- يستخدم آلية الاستيراد الموجودة في القاعدة (الهجرة `0016`) — **لا يحتاج أي هجرة جديدة**. لا يُحفظ شيء قبل «اعتماد».
- التحقق: typecheck + build نظيفان.

### ميزة — المستأجرون والعقود والحالة المالية
- **المستأجرون** `/app/tenants`: قائمة + إضافة مستأجر (اسم، نوع، هوية/إقامة، جوال، بريد).
- **العقود** `/app/contracts`: إنشاء عقد كمسودة (وحدة + مستأجر + مدة + إيجار سنوي + دورية + تأمين + رسوم).
- **تفاصيل العقد** `/app/contracts/[id]`: تفعيل العقد، وجدول الاستحقاقات، وتسجيل الدفعات، والحالة المالية المشتقّة (مدفوع/متأخر/متبقّي).
- **بيانات** — هجرة `0019`: دالتان ذرّيتان في القاعدة:
  - `activate_contract` — يحوّل المسودة إلى نشط، ويولّد جدول الاستحقاقات حسب الدورية (تقسيم متساوٍ مع الباقي على آخر دفعة)، والضريبة (سكني 0%، تجاري 15%)، ويجعل الوحدة «مؤجرة».
  - `record_charge_payment` — ينشئ دفعة ويطابقها على الاستحقاق (بحد أقصى المتبقّي؛ الزائد رصيد للمستأجر).
- التحقق: 36/36 (لا انحدار) + اختبار مخصّص للدالتين (توليد الاستحقاقات، الضريبة، التفعيل، الدفعات الجزئية/الزائدة) + typecheck/build نظيفان.
- ⚠️ **يتطلب تطبيق هجرة `0019` على القاعدة الحيّة** ليعمل التفعيل وتسجيل الدفعات (إضافة/عرض المستأجرين والعقود تعمل بدونها).

### ميزة — أول شاشات سجل الحقيقة: العقارات والوحدات
- **العقارات** `/app/properties`: عرض قائمة عقارات المنشأة النشطة وإضافة عقار (اسم، نوع، مدينة، حي، رقم صك). المالك الافتراضي = المالك الذاتي للمنشأة.
- **تفاصيل العقار** `/app/properties/[id]`: بيانات العقار + جدول وحداته + إضافة وحدة (رقم، حالة، دور، مساحة، غرف، دورات مياه).
- شريط تنقّل موحّد للتطبيق (الرئيسية · العقارات) واسم المنشأة النشطة، مع حماية المسارات (تحويل لغير المسجّل).
- كل القراءة/الكتابة تمرّ عبر RLS بسياق `x-active-org` — لا تظهر إلا بيانات المنشأة النشطة.
- التحقق: typecheck + build نظيفان.

### بنية — النشر على Vercel وتشغيل البيئة الحيّة
- ضبط `NEXT_PUBLIC_SUPABASE_URL` و`NEXT_PUBLIC_SUPABASE_ANON_KEY` في مشروع Vercel `aqar` (production + preview) وإطلاق نشر جديد.
- **الرابط الحيّ: https://aqar-eta.vercel.app** — فحص `/api/health` يُرجع `ok:true` (الاتصال بـ Supabase سليم، RLS تعمل).
- تعطيل حماية النشر (Vercel Authentication) مؤقتاً ليكون الرابط قابلاً للعرض/المشاركة. **يُنصح بإعادة تفعيلها قبل إدخال بيانات حقيقية** (Vercel ← المشروع ← Settings ← Deployment Protection).
- تطبيق الهجرتين `0017` و`0018` على القاعدة، وتفعيل مزوّد الجوال + رقم تجريبي — أصبح تسجيل الدخول يعمل على الرابط الحيّ.

### ميزة — المرحلة 2ب: تسجيل الدخول بالجوال (OTP) والمنشآت — `436f23a`
- شاشة `/login`: إدخال رقم الجوال ← استقبال رمز التحقق ← الدخول (عبر Supabase Auth، واجهة عربية RTL).
- شاشة `/app` المحمية: قائمة منشآتك، إنشاء منشأة جديدة، التبديل بين المنشآت، تسجيل الخروج.
- **بيانات** — هجرة `0017`: ربط الهوية بحساب المصادقة (إنشاء ملف الهوية تلقائياً عند أول دخول).
- **بيانات** — هجرة `0018`: تمكين المستخدم من رؤية منشآته (لعمل المبدّل) دون المساس بعزل البيانات.
- **قرار** — اعتماد **Supabase Auth** لتسجيل الدخول بالجوال (بدل نظام OTP المخصّص)، مع إمكانية توصيل مزوّد رسائل سعودي لاحقاً عبر SMS Hook.
- التحقق: بناء نظيف، `/login` تعمل، `/app` تحوّل لغير المسجّل، فحص الاتصال `ok:true`، واختبارات القاعدة 36/36 + 3/3.
