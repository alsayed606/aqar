## 2026-07-22

### ميزة — الفاتورة الضريبية (ZATCA — المرحلة الأولى)
- **إصدار فاتورة لكل استحقاق** من صفحة العقد (زر «إصدار» بجانب كل استحقاق؛ ومن أصدر تظهر بجانبه رقم فاتورته).
- **`/app/invoices`**: قائمة الفواتير (رقم، تاريخ، مشترٍ، ضريبة، إجمالي، نوع) مع رابط لكل فاتورة.
- **`/app/invoices/[id]`**: فاتورة قابلة للطباعة (عربي RTL): بيانات المورّد (اسم + س.ت + الرقم الضريبي)، بيانات المشتري، بنود الفاتورة، الإجماليات (غير شامل/الضريبة/الإجمالي)، المبلغ **كتابةً**، و**رمز QR** بالحقول الخمسة (اسم البائع، الرقم الضريبي، التاريخ، الإجمالي، الضريبة) بترميز TLV/Base64.
- **مرونة التسجيل الضريبي:** المورّد على فاتورة الإيجار هو **المالك** (المكتب وكيل)، أو **المنشأة** للعقارات الذاتية. إن كان له رقم ضريبي → **فاتورة ضريبية مبسطة + QR**؛ وإلا → **فاتورة عادية** بلا ضريبة/QR. **سكني معفى (0%) مع بيان الإعفاء**، **تجاري 15%**. أُضيف حقل الرقم الضريبي/السجل للمالك (نموذج في صفحة المالك).
- **بيانات** — هجرة `0023`: جدولا `invoice` و`invoice_line`، ودالة `issue_invoice(charge)` (لقطة ثابتة للمورّد/المشتري، بنود من الاستحقاق، إجماليات، نوع الفاتورة، ومنع تكرار الفاتورة لنفس الاستحقاق)، وترقيم متسلسل بلا فجوات لكل (منشأة، سنة) `INV-YYYY-NNNNN`. الفاتورة توثّق التوريد وهي **مستقلّة عن السداد** (تُصدَر بكامل المبلغ بصرف النظر عن الدفع الجزئي).
- أداة `lib/zatca.ts` لبناء حمولة QR، و`qrcode` لتوليد الرمز.
- التحقق: اختبار الإصدار **20/20** (VAT 15%/الإعفاء السكني، اللقطة، الترقيم، منع التكرار، فاتورة عادية مقابل ضريبية، المورّد الذاتي = المنشأة) + اختبار QR TLV 6/6 (فكّ الترميز يطابق الحقول، العربية محفوظة) + السويت الرئيسي 36/36 + typecheck/build نظيفان.
- ⚠️ **يتطلب تطبيق هجرة `0023`** لتفعيل الفواتير. الفاتورة الضريبية بالمرحلة الأولى؛ ربط المرحلة الثانية (المقاصة/الإبلاغ والتوقيع التشفيري) يحتاج تسجيل المنشأة في «فاتورة» لاحقاً.

### ميزة — سند القبض (إثبات التحصيل، قابل للطباعة)
- **لكل دفعة مستلمة سند قبض مرقّم** — إثبات استلام المبلغ (خاصة للكاش، وربط التحويل البنكي بالعقد/الاستحقاق).
- **`/app/receipts`**: قائمة كل السندات (رقم السند، التاريخ، المستلم منه، المبلغ، الطريقة) مع رابط لكل سند.
- **`/app/receipts/[id]`**: سند قبض قابل للطباعة (A4، عربي RTL): اسم المنشأة + س.ت/الرقم الضريبي، رقم السند وتاريخه، المستلم منه وهويته، المبلغ رقماً و**كتابةً (تفقيط)**، طريقة الدفع والمرجع، وبيان «وذلك عن» (العقد/الاستحقاقات المسدَّدة)، وخانتا المستلم والختم. زر طباعة يخفي واجهة التطبيق.
- **صفحة العقد**: قسم جديد «الدفعات المستلمة» يربط كل دفعة بسندها.
- **بيانات** — هجرة `0022`: عدّاد مستندات لكل منشأة (`org_counter` + `next_counter`) وترقيم **متسلسل بلا فجوات لكل (منشأة، سنة)** يُسند تلقائياً على كل دفعة (trigger) بصيغة `RV-YYYY-NNNNN`، مع تعبئة رجعية للدفعات الموجودة. السند يوثّق **الاستلام** وهو مستقلّ عن الفاتورة الضريبية (توثّق التوريد+الضريبة).
- أداة **التفقيط** العربية (`lib/tafqit.ts`) لتحويل المبلغ إلى كلمات على السند.
- التحقق: اختبار مخصّص للترقيم 9/9 (تسلسل لكل منشأة، إعادة ضبط سنوي، منع التكرار، التعبئة الرجعية) + السويت الرئيسي 36/36 (لا انحدار) + typecheck/build نظيفان.
- ⚠️ **يتطلب تطبيق هجرة `0022`** لتظهر أرقام السندات وتُطبع.

### ميزة — لوحة المؤشرات (الصفحة الرئيسية للمكتب)
- **الصفحة الرئيسية `/app`** أصبحت **لوحة مؤشرات** تظهر عند كل دخول:
  - **نسبة الإشغال** (مؤجرة/إجمالي الوحدات) بشريط تقدّم + عدد الشاغرة.
  - **تحصيل الشهر** (ر.س) مع اتجاه مقارنة بالشهر الماضي (▲/▼).
  - **المتأخرات** (ر.س) وعدد الاستحقاقات المتأخرة (بلون تنبيه عند وجودها).
  - **إجمالي المستحق** غير المسدَّد على المستأجرين.
  - شريط نشاط: **العقود النشطة**، **ما ينتهي خلال 60 يوماً** (رادار التجديد)، عدد العقارات، عدد المستأجرين — كلها روابط سريعة.
- المؤشرات العددية (الإشغال/العقود/العقارات) تُحسب عبر **استعلامات عدّ خفيفة** (بلا نقل صفوف) وتعمل فوراً. البطاقات المالية تعتمد على دالة القاعدة.
- **بيانات** — هجرة `0021`: دالة `dashboard_finance()` تُرجع صفاً واحداً (المستحق، المتأخرات، عدد المتأخرات، تحصيل الشهر، تحصيل الشهر السابق) — حدود الشهر بتوقيت الرياض. SECURITY INVOKER (يسري RLS ونطاق المنشأة النشطة).
- تدرّج آمن: إن لم تُطبَّق `0021` تظهر البطاقات المالية برسالة «طبّق هجرة 0021» بدل الخطأ، وباقي اللوحة يعمل.
- التحقق: اختبار مخصّص للدالة 7/7 (المستحق 9,000,000؛ متأخرات موجودة؛ تحصيل الشهر 3,000,000؛ فصل الشهر السابق 500,000) + typecheck/build نظيفان.
- ⚠️ **يتطلب تطبيق هجرة `0021`** لتظهر بطاقتا التحصيل والمتأخرات.

### ميزة — الملّاك وكشوف الحسابات
- **الملّاك** `/app/owners`: قائمة + إضافة مالك (اسم، نوع، هوية/سجل، جوال، آيبان، بنك).
- **ربط العقار بمالك:** اختيار المالك عند إنشاء العقار، وإعادة تعيينه من صفحة العقار (بدل المالك الذاتي الافتراضي).
- **تفاصيل المالك** `/app/owners/[id]`: عقاراته، **نسبة أتعاب الإدارة** (تُحفظ كاتفاقية إدارة)، و**كشف الحساب** لفترة: المُحصَّل − الأتعاب = الصافي للمالك، مع المتبقّي على المستأجرين.
- **بيانات** — هجرة `0020`: دالة `owner_statement(owner, from, to)` تجمع (لكل عقار) المُحصَّل خلال الفترة، والأتعاب (من اتفاقية النسبة)، والصافي، والمتبقّي. SECURITY INVOKER (يسري RLS ونطاق العقارات).
- التحقق: اختبار مخصّص (تحصيل 3,000,000 → أتعاب 5% = 150,000 → صافي 2,850,000 → متبقٍّ 9,000,000؛ خارج الفترة = 0) + typecheck/build نظيفان.
- ⚠️ **يتطلب تطبيق هجرة `0020`** لتظهر كشوف الحساب.

### ميزة — المستأجرون والعقود والحالة المالية

- **تفاصيل العقد** `/app/contracts/[id]`: تفعيل العقد، وجدول الاستحقاقات، وتسجيل الدفعات، والحالة المالية المشتقّة (مدفوع/متأخر/متبقّي).
- **بيانات** — هجرة `0019`: دالتان ذرّيتان في القاعدة:

- التحقق: 36/36 (لا انحدار) + اختبار مخصّص للدالتين (توليد الاستحقاقات، الضريبة، التفعيل، الدفعات الجزئية/الزائدة) + typecheck/build نظيفان.
- ⚠️ **يتطلب تطبيق هجرة `0019` على القاعدة الحيّة** ليعمل التفعيل وتسجيل الدفعات (إضافة/عرض المستأجرين والعقود تعمل بدونها).
